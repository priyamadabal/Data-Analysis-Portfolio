Hyper Parameter Tuning

1.K-nearest Neighbors (KNN)

We employ a Pipeline to combine feature selection and grid search for tuning KNN hyperparameters through cross-validation. The same Pipeline approach will be used for Naive Bayes and Decision Trees

The hyperparameters for KNN are:

  • Number of neighbors (n_neighbors)
  • Distance metric (p)
In[21]:
# custom function for RFI feature selection inside a pipeline
# here we use n_estimators=100
class RFIFeatureSelector(BaseEstimator, TransformerMixin):
    
    # class constructor 
    # make sure class attributes end with a "_"
    # per scikit-learn convention to avoid errors
    def __init__(self, n_features_=10):
        self.n_features_ = n_features_
        self.fs_indices_ = None

    # override the fit function
    def fit(self, X, y):
        from sklearn.ensemble import RandomForestClassifier
        from numpy import argsort
        model_rfi = RandomForestClassifier(n_estimators=100)
        model_rfi.fit(X, y)
        self.fs_indices_ = argsort(model_rfi.feature_importances_)[::-1][0:self.n_features_] 
        return self 
    
    # override the transform function
    def transform(self, X, y=None):
        return X[:, self.fs_indices_]

# custom function to format the search results as a Pandas data frame
def get_search_results(gs):

    def model_result(scores, params):
        scores = {'mean_score': np.mean(scores),
             'std_score': np.std(scores),
             'min_score': np.min(scores),
             'max_score': np.max(scores)}
        return pd.Series({**params,**scores})

    models = []
    scores = []

    for i in range(gs.n_splits_):
        key = f"split{i}_test_score"
        r = gs.cv_results_[key]        
        scores.append(r.reshape(-1,1))

    all_scores = np.hstack(scores)
    for p, s in zip(gs.cv_results_['params'], all_scores):
        models.append((model_result(s, p)))

    pipe_results = pd.concat(models, axis=1).T.sort_values(['mean_score'], ascending=False)

    columns_first = ['mean_score', 'std_score', 'max_score', 'min_score']
    columns = columns_first + [c for c in pipe_results.columns if c not in columns_first]

    return pipe_results[columns]

2.(Gaussian) Naive Bayes (NB)

We implement a Gaussian Naive Bayes model, optimizing the var_smoothing parameter (a variant of Laplace smoothing) due to the lack of prior information about our dataset. The default value for var_smoothing is (10^{-9}). We perform grid search over a log space of powers of 10, ranging from (10^{-10}) to (10^{-1}), using 200 different values generated by NumPy. However, for efficiency, we conduct a random search over just 20 of these values. Since the Gaussian Naive Bayes model assumes that each descriptive feature follows a Gaussian distribution, we apply a power transformation to the input data before fitting the model.

# Define the neural network model
def create_model(input_dim, layer1_units=4, layer2_units=4, optimizer='adam'):
    model = Sequential()
    model.add(Dense(layer1_units, input_dim=input_dim, activation='relu'))
    model.add(Dropout(0.1))
    model.add(Dense(layer2_units, activation='relu'))
    model.add(Dropout(0.1))
    model.add(Dense(1, activation='sigmoid'))
    if optimizer == 'adam':
        optimizer = Adam()
    elif optimizer == 'rmsprop':
        optimizer = RMSprop()
    model.compile(loss='binary_crossentropy', optimizer=optimizer, metrics=['accuracy'])
    return model

# Create a custom wrapper
class KerasClassifierWrapper(BaseEstimator, ClassifierMixin):
    def __init__(self, input_dim, layer1_units=4, layer2_units=4, optimizer='adam', epochs=10, batch_size=10, verbose=0):
        self.input_dim = input_dim
        self.layer1_units = layer1_units
        self.layer2_units = layer2_units
        self.optimizer = optimizer
        self.epochs = epochs
        self.batch_size = batch_size
        self.verbose = verbose
        self.model_ = None
        self._create_model_fn = lambda: create_model(self.input_dim, self.layer1_units, self.layer2_units, self.optimizer)

    def fit(self, X, y, **fit_params):
        self.model_ = self._create_model_fn()
        self.model_.fit(X, y, epochs=self.epochs, batch_size=self.batch_size, verbose=self.verbose, **fit_params)
        return self

    def predict(self, X):
        return (self.model_.predict(X) > 0.5).astype("int32")

    def score(self, X, y):
        return accuracy_score(y, self.predict(X))

Critique and Limitations

Limitations:

  1. Feature Selection: If the feature set used for predictive modeling is not comprehensive or relevant, it may lead to suboptimal performance. Ensuring the inclusion of clinically relevant features and effective feature selection techniques could enhance the model's predictive accuracy.

  2. Imbalanced Data: Imbalanced datasets, where one class (e.g., benign (0) or malignant(1)) is significantly more prevalent than the other, can bias model predictions towards the majority class. Implementing techniques such as oversampling, undersampling, or utilizing specialized algorithms for handling imbalanced data could mitigate this issue.

  3. Model Interpretability: Decision Trees and Neural Networks, while effective in predictive modeling, often lack interpretability compared to Naive Bayes and KNN. Understanding the rationale behind model predictions is crucial, especially in medical contexts where interpretability is essential for decision-making.

  4. Hyperparameter Tuning: The performance of machine learning models is sensitive to hyperparameters, and suboptimal hyperparameter values can impact model performance. Exhaustive hyperparameter tuning may be computationally expensive and prone to overfitting, necessitating careful selection and validation of hyperparameters.

Strengths:

  1. Diverse Modeling Approaches: By employing a variety of classifiers such as KNN, Decision Trees, Naive Bayes, and Neural Networks, your approach harnesses the strengths of different algorithms, enabling a comprehensive exploration of the data landscape and potentially capturing complex relationships.

  2. Robustness: Utilizing multiple classifiers enhances the robustness of your predictive modeling approach. If one model underperforms due to inherent biases or limitations, others may compensate, leading to more reliable predictions overall.

  3. Scalability: KNN, Decision Trees, Naive Bayes, and Neural Networks are scalable algorithms that can handle datasets of varying sizes. This scalability facilitates the application of your predictive modeling approach to large-scale datasets encountered in clinical settings.

  4. Generalizability: A well-constructed predictive modeling approach can generalize to unseen data, providing valuable insights and predictions beyond the training dataset. Cross-validation techniques and rigorous evaluation metrics help assess the generalizability of your models.

Conclusion

Our dataset contains various patient attributes, including demographic data, sexual behavior, smoking habits, contraceptive usage, history of sexually transmitted diseases (STDs), and diagnostic outcomes related to cervical cancer. This project was divided into two main phases: data cleaning and predictive modeling.

Phase 1: Data Cleaning and Exploration In the initial phase, our primary goal was to prepare the dataset for analysis. We identified the biopsy result as our target variable. Through meticulous data cleaning, we removed null values and inconsistencies to ensure the reliability of our analyses. We then explored the relationships between the variables to understand how they influence each other. Visualizations provided valuable insights into trends, such as the prevalence of cervical cancer risk factors across different demographic groups and the impact of certain behaviors on disease incidence. This exploratory analysis laid a strong foundation for the predictive modeling phase.

Phase 2: Predictive Modeling In the second phase, we focused on predictive modeling to determine the likelihood of cervical cancer based on the patient attributes. We identified the most significant features contributing to the prediction using feature importance techniques, finding that the Hinselmann result, Schiller result, and age were the most important features in predicting the correct biopsy result. We then split the dataset into training and testing sets in a 70:30 ratio to evaluate our models effectively. We employed several machine learning algorithms, including K-Nearest Neighbors (KNN), Naive Bayes, Decision Trees, and Neural Networks.

After training and testing these models, we performed t-tests to compare their performance based on the ROC AUC scores. Our findings indicated that:

Naive Bayes had the highest mean ROC AUC score of 0.930, significantly outperforming both Decision Trees (0.802) and KNN (0.859). KNN also performed well but did not show a statistically significant difference in mean ROC AUC score when compared to Neural Networks, both averaging around 0.750. Decision Trees, while effective, were outperformed by both Naive Bayes and KNN. Neural Networks did not significantly outperform any of the other models in terms of ROC AUC score.

Given these results, the Naive Bayes classifier is the most effective model for predicting cervical cancer biopsy results in this dataset, followed by KNN and Decision Trees. The identification of key features, such as the Hinselmann result, Schiller result, and age, further enhances the predictive accuracy of our models. These findings will greatly assist in predicting cancer biopsy results, providing valuable support for clinical decision-making and potentially improving patient outcomes. This project highlights the importance of data preprocessing and the value of comparative analysis in selecting the most suitable predictive model for medical diagnostics.
